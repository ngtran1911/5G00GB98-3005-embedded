Slide 3: Key Functions (1/2)

1. checkButton()

Reads button input, detects press, cycles send mode

How: Polls pin D2 each loop. Detects falling edge (HIGH→LOW transition). Ignores bounces within 200ms using a timestamp check. On valid press, cycles sendMode through 0→1→2→0 and resets the send timer so the countdown restarts from the new mode.

2. readHumidity()

Measures humidity sensor signal and returns averaged RH%

How: Uses pulseIn() to measure the HIGH and LOW duration of the sensor's square wave (50ms timeout). Calculates frequency from the period: freq = 1,000,000 / (hi + lo). Converts to RH% using the datasheet formula RH = -0.06 × freq + 514, then passes through the sliding average. Returns -1 if no signal detected.

3. updateAverage()

Smooths noisy sensor readings using a 10-element circular buffer

How: Maintains a fixed-size array of 10 readings. Each call: subtracts the oldest value from the running total, overwrites it with the new value, adds the new value to the total, and advances the index with wraparound (readIndex = (readIndex + 1) % 10). Returns total / count. This avoids recalculating the sum every time — O(1) per update.

Slide 4: Key Functions (2/2)

4. prefillHumidityBuffer()

Blocks at startup until the humidity sensor responds, then initializes the buffer

How: Runs a do-while loop calling pulseIn() until both HIGH and LOW pulses are detected. Takes the first valid reading and fills all 10 buffer slots with it. Without this, the average would start at 0% and slowly climb to the real value over 10 readings — giving wrong data for the first few seconds.

5. updateDisplay()

Renders sensor data, mode, and countdown on the 20×4 LCD

How: Only called every 500ms (or immediately on mode change) to prevent flicker. Row 0 shows the current mode and seconds until next MQTT send. Rows 1-2 show sensor data based on sendMode — only the data being published is displayed. Row 3 shows the IP address. All lines are padded to 20 characters to overwrite old text.

6. sendMQTT()

Formats sensor data as JSON and publishes to the MQTT broker

How: First checks if the MQTT connection is alive; reconnects if needed. Calls formatPayload() which uses dtostrf() to convert floats to strings and snprintf() to build a JSON string. The JSON structure changes based on sendMode: temperature only, humidity only, or both. Publishes to topic ICT4_out_2020.
